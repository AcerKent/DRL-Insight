## 🌟 Introduction (專案簡介)

**DRL Insight** is a specialized dashboard designed for medical imaging departments and clinical radiology professionals to analyze and visualize **Diagnostic Reference Levels (DRL)**. 

It solves the core pain point of complex software setups and data privacy concerns. By running entirely on the client side (in your web browser), you can directly import Excel reports generated by medical instruments without uploading any sensitive patient data to the cloud. It instantly generates intuitive statistical charts, making local data analysis both secure and effortless.

**DRL Insight** 是一個專為醫學影像科與臨床放射人員設計的「輻射劑量參考水平 (DRL)」分析與視覺化面板。

它解決了繁瑣設定與病患個資隱私的痛點。本系統完全在本地瀏覽器端執行，醫事人員可直接匯入由儀器產生的 Excel 報表，無須將任何敏感資料上傳至雲端，即可在幾秒鐘內產生直觀的統計圖表，讓本地端的數據分析變得既安全又輕鬆。

---

## ✨ Core Features (核心功能特色)

*   **🔒 100% Client-Side Processing (本地端隱私運算)**
    *   **EN:** All data extraction, parsing, and filtering occur entirely within your web browser. No backend server is required, meaning no patient data or hospital reports ever leave your computer.
    *   **TW:** 所有的 Excel 解析與過濾統計皆在瀏覽器端獨立完成，不需連接任何後端伺服器，100% 確保病患個資與醫院機密絕不外流。
*   **📊 Advanced DRL Metrics Visualization (視覺化劑量指標)**
    *   **EN:** Effectively aggregates and overlays `Mean`, `P50` (Median), and `P75` percentiles on interactive histograms to intuitively display radiation exposure deviations.
    *   **TW:** 在動態長條圖上同時結合「平均值」與精準疊加的「P50 中位數」及「P75 百分位」標記點，協助您秒速揪出劑量超標的異常現象。
*   **📈 Time-Series Trend Insights (自動化歷史趨勢分析)**
    *   **EN:** Automatically extracts temporal data from `StudyDate` columns to plot continuous trend lines for both CTDIvol and DLP metric variations over time.
    *   **TW:** 系統會自動抓取檢查日期欄位，並針對不同部位或掃描協定繪製連續的時間曲線，幫助使用者追蹤長期的輻射劑量變化軌跡。

## 📊 Data Requirements (資料格式要求)

> **💡 Smart Parsing:** The system uses fuzzy matching for column headers. Your Excel file (`.xlsx` or `.xls`) must contain at least some of the following columns to generate the respective charts. (系統具備模糊比對功能，您的報表欄位只需包含以下部分關鍵字即可成功解析)

*   **⚡ Dose Metrics (必備劑量欄位)**: 
    *   `CTDIvol`, `Mean CTDIvol (mGy)`
    *   `DLP (mGy.cm)`, `Total DLP`
*   **📑 Categorization (必備分類欄位，至少一項)**: 
    *   `Acquisition Protocol`, `Protocol`, `ExamName`
    *   `Target Region`, `Region`, `BodyPart`
    *   `StudyDescription`, `Description`
*   **📅 Time Series (自由選填日期欄位)**: 
    *   `StudyDate`, `Examination Date`, `Date`. 
    *   Supports YYYY/MM/DD and DICOM plain format (`20240315`). (支援標準日期格式與醫療儀器常見的 YYYYMMDD 無符號格式)。

## 🚀 User Guide (操作說明 - For Clinical Users)

### **EN**
1. **Export Data:** Export your CT dose reports from your hospital's system in Excel format (`.xlsx`).
2. **Open Dashboard:** Navigate to the DRL Insight web page.
3. **Upload File:** Drag and drop your Excel file into the designated upload zone on the left sidebar.
4. **Interactive Filtering:** Use the collapsible tree checklist on the left to filter the data by Protocol, Region, or Description. The charts and metrics will instantly update in real-time.

### **TW**
1. **匯出資料:** 從院內系統將 CT 輻射劑量報告匯出為 Excel (`.xlsx`) 格式。
2. **開啟網頁:** 進入 DRL Insight 網頁介面。
3. **上傳報表:** 點擊左側上傳區塊，或直接將您的 Excel 檔案拖拉放進虛線框內。
4. **互動統計:** 檔案讀取後，利用左下角的「樹狀多選清單」勾選關注的掃描部位或協定，右側的所有統計指標、直方圖、趨勢圖將會無縫連動運算，無須點擊任何額外的分析按鈕。

## 💻 Developer Guide (開發者與部屬指南)

### 1️⃣ Local Execution (本地執行測試)
**EN:** You don't need Node.js or complex server setups. A simple HTTP server is enough.
1. Open your terminal and navigate to the `frontend` folder.
2. Run a simple local server: `python -m http.server 8000`
3. Open `http://localhost:8000` in your browser.

**TW:** 不需 Node.js 也不用處理任何相依套件，只需一個最簡單的網頁伺服器。
1. 開啟終端機 (Terminal) 並進入專案內的 `frontend` 資料夾。
2. 執行指令啟動伺服器：`python -m http.server 8000`
3. 於瀏覽器開啟 `http://localhost:8000` 即可使用。

### 2️⃣ Deployment to GitHub Pages (部屬到 GitHub Pages)
**EN & TW: 透過 GitHub Actions 自動部署指南**

本專案已內建 `.github/workflows/deploy.yml` 工作流。只要您將程式碼推送到 GitHub，系統就會自動把 `frontend` 資料夾部屬到 GitHub Pages 上。

> ⚠️ **Important Settings Requirement (重要權限設定)**
> Before pushing your code, please ensure your GitHub Repository settings are configured correctly to avoid Action failures:
> 推送程式碼前，請務必按照以下步驟開放權限，否則自動部屬將會失敗：

1. 到您 GitHub 專案頁面上方的 **Settings**。
2. 點擊左側選單的 **Actions** -> **General**。
3. 往下滑找到 **Workflow permissions**，選擇 `Read and write permissions`，並點擊 `Save`。
4. 點擊左側選單的 **Pages**。
5. 在 **Build and deployment** 底下的 **Source** 選擇 `GitHub Actions`。
6. 設定完成後，只要執行 `git push origin main`，您的專案就會在數分鐘內變成線上公開網頁了！
